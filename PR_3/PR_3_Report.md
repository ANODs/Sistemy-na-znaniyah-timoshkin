## 1 Задание к работе
1.1. Разработать программу на языке высокого уровня Python для составления нечеткой продукционной системы и прямого вывода:
1) Программа предоставляет интерфейс для добавления требуемого числа правил, включая их условия, заключения и коэффициенты уверенности (CF правила). Предусмотрено не менее 3-х видов условий: простое, составное И (min), составное ИЛИ (max).
2) Полученная нечеткая продукционная система выводится на экран.
3) На основе введенных пользователем исходных фактов с их CF программа реализует процесс прямого вывода и выводит результаты расчетов итогового CF по каждому правилу.

1.2. Продемонстрировать процесс тестирования:
1) Добавление не менее чем 5 правил, связанных с темой планируемой ВКР в магистратуре (Тема: **Выбор архитектуры базы данных**).
2) Привести два примера прямого вывода с необходимыми расчетами (перемножение CF условия на CF правила).

**Тема ВКР:** Выбор архитектуры базы данных. Нечеткая логика позволяет определить наиболее подходящую систему при нечетких (вероятностных) требованиях, когда разные требования конфликтуют или соблюдаются лишь частично.

**Результат выполнения программы №1 (Первый пример):**
```text
============================================================
ПРАКТИЧЕСКАЯ РАБОТА №3: НЕЧЕТКИЕ ПРОДУКЦИОННЫЕ СИСТЕМЫ
Тема ВКР: Выбор архитектуры базы данных
============================================================
Использовать готовые нечеткие правила для БД? (y/n): y
Ваша нечеткая продукционная система:
Правило 1: ЕСЛИ Сложные ACID транзакции ТО Использовать Реляционную БД (PostgreSQL) (CF правила = 0.95)
Правило 2: ЕСЛИ Высокая нагрузка на чтение AND Высокая нагрузка на запись ТО Использовать NoSQL (Cassandra/MongoDB) (CF правила = 0.85)
Правило 3: ЕСЛИ Графовые связи OR Неструктурированные данные ТО Использовать Графовую БД (Neo4j) (CF правила = 0.8)
Правило 4: ЕСЛИ Высокая скорость поиска текста ТО Использовать Поисковый движок (Elasticsearch) (CF правила = 0.9)
Правило 5: ЕСЛИ Временные ряды AND Аналитика в реальном времени ТО Использовать БД временных рядов (InfluxDB) (CF правила = 0.85)

--- ПРОЦЕСС ПРЯМОГО ВЫВОДА ---
Введите исходные факты и их CF в формате 'Факт1:CF1, Факт2:CF2'
Пример: Сложные ACID транзакции:0.9, Высокая нагрузка на запись:0.8
Ваш ввод: Сложные ACID транзакции: 0.8, Временные ряды: 0.9, Аналитика в реальном времени: 0.7

Результаты вывода по каждому правилу:
Правило 1: АКТИВНО (CF = 0.76)
Правило 2: НЕ АКТИВНО (CF = 0.00)
Правило 3: НЕ АКТИВНО (CF = 0.00)
Правило 4: НЕ АКТИВНО (CF = 0.00)
Правило 5: АКТИВНО (CF = 0.59)

ИТОГОВЫЕ РЕКОМЕНДАЦИИ (по убыванию CF):
- Использовать Реляционную БД (PostgreSQL) с уверенностью 0.76
- Использовать БД временных рядов (InfluxDB) с уверенностью 0.59
```
*(Пример фактов: "Сложные ACID транзакции: 0.8, Временные ряды: 0.9, Аналитика в реальном времени: 0.7")*
Расчеты: Для правила с И берется `min(0.9, 0.7) = 0.7`, затем `0.7 * CF правила` и т.д.
